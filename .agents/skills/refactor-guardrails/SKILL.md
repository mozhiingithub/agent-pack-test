# 重构约束

## 前置检查（不满足不动手）

- 人已公告"X 模块重构窗口"，修复方已知会；
- 重构已拆成 1~2 天/片的切片，每片 spec 声明"对外行为不变"。

## 规则

1. 每片独立走 `skills/feat-workflow` 全流程（分支、同步、实测、合并），不攒大重构分支；
2. 验收红线：既有测试**原样**全绿；spec 不许出现"顺带优化了行为"；
3. 需要改行为 → 拆成独立 feat/fix 分支单独走流程，严禁与重构混在同一 commit；
4. 窗口内模块出 bug → 修复线先行，当天 `git merge main` 吸收修复后继续；
5. 绝不在重构分支里顺手修 bug、顺手改样式。

## 停手信号

切片导致既有测试需要修改才能通过 → 说明行为变了，停下来向人报告，重新拆片。
