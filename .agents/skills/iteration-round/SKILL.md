# 实测失败迭代轮

## 步骤

1. 读人回传的复现步骤、完整日志、截图；信息不足以定位就**先向人要**，不盲改；
2. 在当前同一分支上修改，本地检查全绿；
3. 按 `skills/sync-package` 生成新序号 sync 包，交人；
4. 向人汇报：改了什么、为什么认为这次能过、还需要哪些信息（若有）。

## 红线

- 不新建分支、不合 main、不推倒重来（换方案需人同意）；
- 每轮改动尽量小，便于再次失败时归因；
- 同一处失败两轮以上：停下来向人说明当前假设，请求更多环境信息（对照"外网环境差异清单"：数据库版本、浏览器、网络策略等）。
